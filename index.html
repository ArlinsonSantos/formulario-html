PERGUNTAS

a) Na tag < input > com atributo type="text", para que serve o atributo "placeholder"?

O atributo placeholder exibe um texto de dica dentro do campo de entrada até que o usuário comece a digitar. Ele é útil para fornecer uma orientação sobre o tipo de dado que deve ser inserido.

No exemplo: "< input type="text" placeholder="Digite seu nome"> . Neste exemplo, o texto "Digite seu nome" será exibido até que o usuário insira algum texto no campo.

b) Na tag < input> com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

O atributo checked define se a opção do tipo radio ou checkbox está pré-selecionada quando a página carrega. Ele marca a opção como selecionada por padrão.

c) Na tag < input>, para que serve o atributo "value"?

O atributo value especifica o valor que será enviado ao servidor quando o formulário for submetido. Para tipos de entrada como texto, ele define o valor inicial; para tipos como radio ou checkbox, ele define o valor associado àquele botão.

d) Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

O atributo name deve ser utilizado com o mesmo valor em todos os botões de rádio do grupo. Isso garante que apenas um deles possa ser selecionado de cada vez.

e) Para que serve a tag < label>? Para que serve o atributo "for" nessa tag?

A tag < label> serve para associar um rótulo a um campo de entrada de formulário, melhorando a acessibilidade. O atributo for vincula o rótulo ao campo específico, correspondendo ao valor do atributo id do elemento de entrada. Quando o rótulo é clicado, o campo correspondente ganha o foco ou é ativado (por exemplo, uma caixa de seleção é marcada).

f) Na tag < option>, para que serve o atributo "selected"?

O atributo selected marca uma opção dentro de uma lista suspensa como a opção padrão selecionada quando a página é carregada.

g) Nas tags de formulário, para que serve o atributo "required"?

O atributo required indica que o campo é obrigatório. O formulário não pode ser enviado se o campo marcado como "required" estiver vazio.

h) Para que serve a tag < textarea>?

A tag < textarea> é usada para criar um campo de texto de várias linhas, permitindo que o usuário insira um texto mais longo do que em um campo de entrada comum de uma única linha.

i) Na tag < form>, para que serve o atributo "action"?

O atributo action especifica a URL para a qual os dados do formulário serão enviados quando o formulário for submetido. Ou seja, indica o destino que processará os dados enviados pelo formulário, como um servidor ou uma página específica.

j) Na tag < form>, para que serve o atributo "method"?

O atributo method define o método HTTP a ser usado ao submeter o formulário. Os valores mais comuns são GET (os dados são anexados à URL) e POST (os dados são enviados no corpo da requisição).

k)Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag < form>?

O valor do atributo method deve ser POST para evitar que os dados, como uma senha, apareçam na URL.

l) Qual valor deve ser informado no atributo "method" na tag < form> em um formulário que será usado para fazer upload de arquivos?

O valor do atributo method deve ser POST, e o atributo enctype="multipart/form-data" deve ser usado para permitir o envio de arquivos.

m) O que as tags < fieldset> e < legend> produzem?

A tag < fieldset> agrupa elementos relacionados em um formulário, criando uma borda ao redor. A tag < legend> define um título para o grupo de elementos dentro do < fieldset>.
